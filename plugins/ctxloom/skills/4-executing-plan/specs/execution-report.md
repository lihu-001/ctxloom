# 执行报告格式

执行完成后，在任务活文档 `.ctxloom/tasks/<功能>.md` 末尾追加执行报告。

## 标准格式

```markdown
---

## 执行报告

- **执行结果**：完成（所有步骤已执行并验证通过）
- **执行分支**：feat/coupon
- **执行时间**：2026-06-25 14:30 ~ 16:45
- **改动统计**：新增 8 个文件，修改 5 个文件，共 +850 -120 行

### 步骤执行情况

| 步骤 | 状态 | 测试策略 | 验证结果 |
|---|---|---|---|
| 步骤 1：创建 Coupon 实体类 | ✓ 完成 | test-first | 先红后绿，单测通过 |
| 步骤 2：实现 CouponMapper | ✓ 完成 | test-first | 先红后绿，单测通过 |
| 步骤 3：实现 CouponService | ✓ 完成 | test-first | 先红后绿，单测通过 |
| 步骤 4：实现 CouponController | ✓ 完成 | 可执行验证 | 接口调用正常，返回符合预期 |

### 整体验证

- **编译**：通过（`mvn clean compile`）
- **单元测试**：通过（新增 12 个测试，全部通过）
- **集成测试**：通过（回归测试套件全部通过）
- **手动验证**：调用 POST /api/coupons 创建优惠券，返回 200 OK

### 待处理事项

- 无
```

## 使用说明

- **执行结果**：完成 / 部分完成（说明剩余步骤）
- **改动统计**：可用 `git diff --stat` 获取
- **步骤执行情况**：每步标注状态（✓ 完成 / ✗ 失败 / ⊙ 跳过）、测试策略、验证结果
- **整体验证**：列出关键验证项及结果
- **待处理事项**：执行中发现的问题、需要后续跟进的点
